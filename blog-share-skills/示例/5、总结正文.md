A2A，全称 Agent-to-Agent Protocol，是面向 AI Agent 之间通信与协作的开放协议。

它解决的核心问题不是“Agent 如何调用工具”，而是“多个 Agent 如何协同工作”。

在多 Agent 系统中，一个复杂任务往往需要多个专业 Agent 分工完成，比如日程管理、订单查询、数据分析、审批、客服等。如果没有统一协议，不同 Agent 之间就需要定制接口、适配数据格式，集成成本高、耦合强，也难扩展。A2A 通过标准化机制，让 Agent 能够发现彼此的能力、交换消息、委派任务、跟踪任务状态，并返回最终结果。

它的核心概念包括：

Agent Card：Agent 的能力名片，描述它能做什么、服务地址、输入输出格式和认证方式。

Message：Agent 之间传递的信息，可以是文本、结构化数据、文件或链接。

Task：用于管理长任务的生命周期，比如处理中、等待输入、完成或失败。

Artifact：任务完成后的交付物，比如报告、表格、文件或方案。

简单记：MCP 解决“Agent 如何使用工具”，A2A 解决“Agent 如何协作”。

A2A 的出现，意味着 AI Agent 正在从单体助手，走向可组合、可协作、可扩展的智能体网络。

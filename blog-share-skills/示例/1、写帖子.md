## 什么是 A2A 协议
A2A 协议，全称 **Agent-to-Agent Protocol**，即智能体到智能体协议。它是一种面向 AI Agent 之间通信与协作的开放协议，主要用于解决不同智能体之间如何相互发现、交换信息、委派任务、跟踪任务状态以及返回执行结果的问题。

在 AI Agent 应用中，单个 Agent 往往难以独立完成复杂任务。一个完整任务可能需要多个专业 Agent 分工协作，例如日程管理 Agent、订单查询 Agent、数据分析 Agent、审批 Agent、客服 Agent 等。A2A 协议的作用，就是为这些不同系统、不同框架、不同厂商实现的 Agent 提供统一的交互标准。

## A2A 协议解决的问题
A2A 主要解决的是 **Agent 之间的互操作性问题**。

在没有统一协议的情况下，不同 Agent 之间如果要协作，通常需要定制接口、适配数据格式、处理通信逻辑和认证逻辑。这会导致集成成本高、系统耦合强、扩展困难。

A2A 通过标准化协议定义，使一个 Agent 能够以统一方式调用另一个 Agent，从而降低多 Agent 系统之间的集成复杂度。

它重点解决以下几个问题：

**第一，Agent 如何发现其他 Agent 的能力。**  
调用方需要知道目标 Agent 能做什么、支持什么输入输出格式、服务地址是什么、是否需要认证。

**第二，Agent 如何进行消息交互。**  
Agent 之间需要传递文本、文件、结构化数据等不同类型的信息。

**第三，Agent 如何协同处理长任务。**  
复杂任务通常不是一次请求就能完成，需要支持任务创建、状态跟踪、过程反馈、用户补充信息以及最终结果返回。

**第四，Agent 如何保护自身内部实现。**  
A2A 关注的是 Agent 之间的外部协作接口，而不是要求暴露 Agent 内部的模型、工具、记忆或执行逻辑。

## A2A 的核心概念
### 1. Agent Card
**Agent Card** 可以理解为 Agent 的能力描述文件，也就是 Agent 的“名片”。

它通常用于描述一个 Agent 的基本信息、能力范围、服务地址、支持的输入输出格式、认证方式等内容。其他 Agent 可以通过读取 Agent Card，判断该 Agent 是否适合处理当前任务。

例如，一个旅行 Agent 的 Agent Card 可能会声明它支持航班查询、酒店推荐、行程规划等能力，同时说明调用它需要哪些参数和认证方式。

### 2. Message
**Message** 是 Agent 之间传递的信息单元。

它不局限于自然语言文本，也可以包含结构化数据、文件、链接、图片等内容。这样可以满足不同业务场景下的多模态和结构化交互需求。

例如，数据分析 Agent 可能返回 JSON 数据，文档 Agent 可能返回 PDF 文件，设计 Agent 可能返回图片或设计稿链接。

### 3. Task
**Task** 是 A2A 中用于表示任务的核心对象。

由于很多 Agent 任务具有较长执行周期，因此 A2A 通过 Task 来管理任务生命周期。一个 Task 可以有不同状态，例如正在处理、等待输入、已完成、失败等。

这使得 Agent 之间不仅可以进行简单问答，也可以处理更复杂的异步任务和多步骤任务。

### 4. Artifact
**Artifact** 指任务执行完成后产生的结果或交付物。

它可以是一份报告、一张表格、一段结构化数据、一个文件、一个计划方案等。Artifact 通常是 Task 的最终输出结果，也可以被后续 Agent 继续使用。

## A2A 与 MCP 的区别
A2A 经常会和 MCP 放在一起讨论，但两者关注的问题不同。

**MCP，Model Context Protocol，主要解决的是 Agent 与工具、数据源、外部系统之间的连接问题。**  
例如让 Agent 访问数据库、调用 API、读取文件、连接企业知识库等。

**A2A 主要解决的是 Agent 与 Agent 之间的协作问题。**  
它关注的是一个 Agent 如何发现另一个 Agent、如何向另一个 Agent 委派任务、如何接收任务状态和结果。

可以简单理解为：

**MCP 解决“Agent 如何使用工具”；A2A 解决“Agent 如何协作”。**

在实际系统中，二者并不是替代关系，而是互补关系。一个 Agent 内部可以通过 MCP 调用工具和数据源，同时通过 A2A 与其他 Agent 协作完成更复杂的任务。

## A2A 的典型工作流程
一个典型的 A2A 协作流程可以分为以下步骤：

1. 调用方 Agent 接收到用户任务；
2. 调用方 Agent 根据任务需求查找合适的目标 Agent；
3. 调用方读取目标 Agent 的 Agent Card，确认其能力、接口和认证方式；
4. 调用方通过 A2A 向目标 Agent 发送 Message 或创建 Task；
5. 目标 Agent 执行任务，并在过程中返回状态更新；
6. 如果需要补充信息，目标 Agent 可以请求调用方或用户继续提供输入；
7. 任务完成后，目标 Agent 返回 Artifact；
8. 调用方 Agent 整合结果，并继续执行后续流程或返回给用户。

这个流程体现了 A2A 的核心价值：让多个 Agent 可以像一个协作网络一样分工完成任务。

## 总结
A2A 协议是面向 AI Agent 之间通信与协作的开放协议。它通过 Agent Card、Message、Task、Artifact 等机制，使不同 Agent 能够发现彼此、交换信息、委派任务、跟踪状态并返回结果。

它解决的核心问题不是单个 Agent 如何调用工具，而是多个 Agent 如何在统一标准下协同工作。

因此，A2A 可以被视为多 Agent 系统的重要基础协议之一。它的出现意味着 AI Agent 正在从单体智能助手，逐步走向可组合、可协作、可扩展的智能体网络。

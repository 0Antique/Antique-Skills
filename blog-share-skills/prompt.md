## SOP操作

输入提示词：

1. 我会给一个主题，让AI收集整理这个主题的信息，并写成一个博文。

```
我是一个博主，做人工智能方向知识分享，请你为【】这个主题生成一个博文。
```

2. 将生成的博文和个人logo，给AI。生成美观简洁的HTML界面，用于分享

```
请你结合我提供的【个人 Logo】和【技术博客文档内容】，为我设计一套适合截图发布到小红书 / 技术博客平台的多页 HTML 知识卡片。

主题：【主题】
标题：【标题】
博客文档内容：
【粘贴博客文档内容】

设计目标：
将技术博客内容重新整理为一组“可截图发布”的知识卡片页面，整体风格要统一、清晰、专业、有记忆点，适合 AI 技术知识分享账号长期使用。

视觉风格要求：
1. 整体风格参考个人 Logo：
   - 暖黄色：用于高亮、标题装饰、重点标签、光晕背景
   - 青蓝色：用于技术感线条、边框、代码块、流程箭头
   - 咖啡棕：用于主标题、正文重点、稳定的品牌识别
   - 白色 / 米白色：作为主背景，保持干净、明亮、适合截图
2. 风格关键词：
   - 温暖科技感
   - 知识卡片感
   - 简洁但不空
   - 适合 AI 博主个人品牌
   - 有轻微插画感，但不要过度花哨
3. 不要使用深色赛博朋克风，不要使用大面积黑底，不要使用过多渐变和阴影。
4. 页面需要有统一的品牌识别：Logo、标题样式、页码、装饰元素、配色体系保持一致。

页面规格：
1. 第 1 页：竖版封面，比例 1:1，适合小红书首图。
2. 第 2 页：横版封面，比例 4:3，适合博客横图或备用封面。
3. 从第 3 页开始：每一页均为 4:3 横版知识卡片。
4. 每一页都要是独立完整的一张图，单独截图也能看懂。
5. 页面之间要有连续性，但不能依赖上一页才能理解本页内容。

内容组织要求：
1. 根据博客文档内容进行提炼、重组和可视化表达。
2. 不要简单复制长段文字，要转化为适合卡片阅读的结构。
3. 每页只讲一个核心点，避免信息过密。
4. 内容结构建议包括：
   - 封面页：标题、副标题、核心一句话、Logo、视觉主元素
   - 概念解释页：用通俗语言解释主题是什么
   - 结构拆解页：拆解核心组成部分
   - 工作流程页：用流程图 / 步骤卡片说明运行逻辑
   - 对比页：和相关概念进行对比
   - 实战页：给出代码、命令、目录结构或操作步骤
   - 总结页：提炼价值、适用场景和一句话结论
5. 技术内容要准确，不要为了美观牺牲严谨性。
6. 如果涉及代码、命令、目录结构，请使用清晰的代码块样式。

版式要求：
1. 使用 HTML + CSS 写成一个完整文件。
2. 所有页面放在同一个 HTML 中，每一页用独立的 section/card 表示。
3. 页面尺寸要固定，便于浏览器渲染后截图。
4. 每一页要有足够留白，不能拥挤。
5. 标题层级清晰：
   - 主标题大而醒目
   - 副标题简洁
   - 正文短句化
   - 重点词用色块或加粗突出
6. 多使用卡片、标签、流程箭头、圆角模块、代码框、图标化元素。
7. 页面底部可以放置页码、账号品牌标识或一句固定 Slogan。

品牌元素要求：
1. Logo 应放在封面显眼位置，后续页面可缩小放在角落。
2. 可以抽象使用 Logo 中的元素：
   - 笑脸圆形：代表友好解释
   - 耳机 / 麦克风：代表 AI 助手、Agent、交互
   - 书本：代表知识分享
   - 灯泡：代表技术灵感
   - 星星 / 光点：代表重点、高亮、智能感
3. 不需要完全复刻 Logo，但整体气质要一致。

配色建议：
- 主色：#F7C948 / #FFD966 / 暖黄色
- 辅色：#2CB7C9 / #3AAFC8 / 青蓝色
- 强调色：#5B2E1E / #6B3A2A / 咖啡棕
- 背景色：#FFF8E8 / #FFFDF7 / 米白色
- 边框色：#E8D8B8 / #BFE7EA
- 正文色：#3B2A22

代码输出要求：
1. 直接输出完整 HTML 文件代码。
2. CSS 写在 style 标签中，不要依赖外部框架。
3. 可以使用 emoji 或简单 SVG 图形作为装饰，但不要过量。
4. HTML 要结构清晰，方便后续修改文案。
5. 不要只给设计建议，必须给出可直接保存为 .html 并在浏览器打开的完整代码。
6. 页面要适合直接截图发布，注意边距、字号、对齐和整体视觉平衡。

显示样式检查：
html生成完成之后，必须要检查所有的页面，确保没有格式混乱情况的出现

最终效果：
生成一套统一品牌风格的技术知识卡片 HTML，让读者一眼知道这是同一个 AI 技术博主的内容。
```
